Prerequisites

Make sure you have:

  • An active Amazon Ads account
  • Claude Desktop installed on your computer
  • Admin access to connect your Amazon Ads account

Follow These Steps:

Generate Your MCP Token

What is an MCP Token?

This is like a password that allows Claude to securely access your Amazon Ads data. Treat it like you would any sensitive credential.

Steps to Generate Token:

  1. Log in to your Marketplace Ad Pros account
  2. Navigate to Settings â†’ API Keys
  3. Click "Generate New Key"
  4. Select key type: mcp-read (recommended for beginners)
  5. Copy the generated token

Security Note

mcp-read: Claude can only view data (recommended to start)
mcp-readwrite: Claude can modify campaigns (use only when you're comfortable)

Configure Claude Desktop

macOS Configuration

Your configuration file is located at:

~/Library/Application Support/Claude/claude_desktop_config.json

Windows Configuration

Your configuration file is located at:

%APPDATA%\Claude\claude_desktop_config.json

Linux Configuration

Your configuration file is located at:

~/.config/Claude/claude_desktop_config.json

Configuration Steps:

  1. Open Finder (Mac) or File Explorer (Windows)
  2. Navigate to the path shown above
  3. Open the file with a text editor (TextEdit, Notepad++, VS Code)
  4. Copy the configuration below and paste it into the file
  5. Save the file
  6. Completely quit Claude Desktop and restart it

Important: Streamable HTTP Method

Use this configuration if you have Claude Desktop with Streamable HTTP support (newer versions):

{
  "mcpServers": {
    "marketplaceadpros": {
      "type": "streamable-http",
      "url": "https://app.marketplaceadpros.com/mcp"
    }
  }
}

Troubleshooting

                    
If you see "no active plan" error: Make sure your Marketplace Ad Pros subscription is active

                    If Claude doesn't show MCP tools: Check that the config file was saved correctly and Claude was fully restarted

                    If you see connection errors: Verify your token is correct in the configuration file
